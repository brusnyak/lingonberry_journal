## For Contributors

When adding development documentation:
1. Use descriptive filenames with dates
2. Keep logs focused on specific features/fixes
3. Include before/after comparisons when relevant
4. Document decisions and rationale
5. Update this README when adding new files

## Changelog Format

When documenting changes, use this format:

```markdown
# Feature Name - Date

## Overview
Brief description of what was changed and why.

## Changes Made
- Bullet list of specific changes
- Include file names and line numbers if relevant

## Testing
- How to test the changes
- Expected behavior

## Known Issues
- Any remaining issues or limitations
```
